Testing de Software

¿Qué es el testing?

Se define como el proceso de ejecutar un programa para detectar errores. Si el testing se realiza de manera correcta, los errores se eliminarían del sistema.

Tipos de testing de software

Testing Manual
Se realiza manual, sin ningún tipo de herramienta de automatización o script. El tester ocupa el rol del usuario final e imita su comportamiento para detectar posibles errores.
Testing Automatizado
El tester escribe scripts y utiliza un software para probar el producto, permite re-ejecutar tests rápida y repetidamente. Incrementa la covertura de código, la precisión y ahorra tiempo y dinero a comparación del testing manual.

Tipos de técnica de testing

Test de caja negra
El tester no tiene acceso al código fuente por lo que el test se realiza sin tomar en cuenta la estructura lógica del programa.
Test de caja blanca
El tester conoce cómo funciona el programa internamente y se asegura que todas las operaciones internas se ejecuten de acuerdo a las especificaciones.

Niveles de testing

Testing Unitario
Generalmente creado por los programadores, prueban la unidad más pequeña de software. En este test, se pueden probar una unidad indivualmente o interrelacionada con otras. Se utilizan inputs de prueba y se observan sus outputs.
Las pruebas unitarias consisten en aislar una parte del código y comprobar que funciona a la perfección. Son pequeños tests que validan el comportamiento de un objeto y la lógica.
El unit testing suele realizarse durante la fase de desarrollo de aplicaciones de software o móviles. Normalmente las llevan a cabo los desarrolladores, aunque en la práctica, también pueden realizarlas los responsables de QA.
 Test de integración
Se testean los módulos como grupo, buscando exponer los defectos de las interacciones de las unidades lógicas entre sí.
Test de regresión / Prueba del sistema
Prueba que todos los componentes del programa funcionen bien a medida que se vayan incorporando nuevas funcionalidades.
Prueba de humo / Test de aceptación
Proceso que determina si el software implementado se encuentra estable o no. La confirmación la otorga el equipo de QA y se prueban las funcionalidades más importantes siendo éste el mínimo set de pruebas.
