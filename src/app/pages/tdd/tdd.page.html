TDD

¿Qué es el TDD?

TDD (Test-driven development o desarrollo dirigido por pruebas) es una metodología de desarrollo de software que consiste en escribir primero los tests y después el código fuente.

Ciclo de desarrollo en el TDD

(Re)escribir un test
Probar que la prueba falle
Desarrollar el código
Correr todos los test
Refactorizar
Este ciclo se lo conoce como rojo - verde- refactor.

Rojo: hacer que la prueba falle.

Verde: hacer que la prueba pase.

Refactor: limpieza y mejoras del código desarrollado.

Ventajas de TDD

  • Mejora la calidad del código
  • Ayuda a entender mejor el desarrollo
  • Es incluso más entendible que una documentación

Desventajas del TDD

  • Toma el doble de tiempo desarrollar los tests y el código
  • Refactorizar un test resulta más complejo
